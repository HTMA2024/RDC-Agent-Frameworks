# Optimization Agent
## Multi-Agent Optimization Framework (Skeleton)

Optimization Agent 是面向“性能 / 质量 / 成本 / 体验”优化类任务的多 Agent 框架**骨架**（占位）。当前仅提供项目入口与约定说明，后续将逐步补全核心 Prompt、指标体系与自动化校验。

## 目标（高层）

- 将优化问题标准化：明确指标（KPI）、基线、约束与回归门槛。
- 建立多角色协作：定位瓶颈 → 设计改动 → 评估收益与风险 → 回归验证 → 产物沉淀。
- 保证结论可追溯：每个优化建议都附带证据、对照与可复现实验。

## 典型输入 / 输出（高层）

- 输入示例：性能 Profile、Trace、指标面板、A/B 数据、资源与依赖图、配置矩阵、成本账单等。
- 输出示例：瓶颈定位报告、优化方案清单（含预期收益/风险/回滚）、实验记录（反事实/A-B）、回归验证结果、可检索知识条目。

## 未来建议目录结构（仅描述，不创建）

参考 `debug-agent` 的分层方式，建议逐步演进到：

- `common/`：平台无关的核心 Prompt、契约与质量门槛（SSOT）
- `platforms/`：不同宿主/插件/工作台的适配层
- `docs/`：指标定义、实验方法、回归策略与报告模板
- `knowledge/`：历史优化案例、实验记录与索引
- `hooks/`：门禁校验（例如：性能回归阈值、指标完整性检查）
