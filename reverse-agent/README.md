# Reverse Agent
## Multi-Agent Reverse / Reconstruction Framework (Skeleton)

Reverse Agent 是面向“逆向 / 还原 / 重建”类任务的多 Agent 框架**骨架**（占位）。当前仅提供项目入口与约定说明，后续将逐步补全核心 Prompt、知识库与平台适配层。

## 目标（高层）

- 将逆向任务拆解为可复用的多角色协作流程（信息采集 → 假设生成 → 验证 → 产物沉淀）。
- 对输入证据、推理链路与输出结论做结构化约束，便于审计、复现与知识复用。

## 典型输入 / 输出（高层）

- 输入示例：二进制/脚本/日志、协议流量、行为轨迹、符号/栈信息、配置与环境差异等。
- 输出示例：可验证的重建结论、关键证据索引、最小复现步骤、候选假设及其反事实验证记录、可检索的知识条目。

## 未来建议目录结构（仅描述，不创建）

参考 `debug-agent` 的分层方式，建议逐步演进到：

- `common/`：平台无关的核心 Prompt、契约与质量门槛（SSOT）
- `platforms/`：不同宿主/插件/工作台的适配层
- `docs/`：工作流规范、产物格式、术语与边界
- `knowledge/`：可检索的知识库与会话产物沉淀
- `hooks/`：质量校验/门禁（可选）
